### 3. 索引与约束策略

已为后续查询与分析设计索引：

- 文章：
  - `source + published_at`
  - `author_id + published_at`
  - `content_hash`
- 交易：
  - `symbol + executed_at`
  - `account_id + executed_at`
  - `article_id + executed_at`
- 行情：
  - `symbol + timeframe + traded_at`
  - `market + traded_at`
  - `source + traded_at`
  - 唯一约束 `(symbol, market, timeframe, traded_at, source)`
- 元数据：
  - `schema_version`
  - `processed_at`

同时增加了关键约束：

- 非空标题、非空正文
- 评论数、阅读数、点赞数等非负
- 交易数量和价格必须大于 0
- 交易方向限定
- OHLC 合理性校验
- 情绪分和置信度范围校验

### 4. 数据验证与异常检测

新增统一验证模块 `DataValidator`，覆盖：

- 文章校验
  - 标题为空
  - 正文过短
  - 发布时间晚于抓取时间
  - 评论数与评论 payload 不一致
  - 缺少 `content_hash`
- 交易校验
  - 成交金额与 `quantity * price` 不一致
  - 成交时间在未来
  - 手续费为 0 的提示
  - 缺少外部 ID 和原始 payload 的弱标识提示
- 行情校验
  - K 线时间在未来
  - 平盘但有成交量
  - 相比前收盘跳空超过 20%
  - 成交量超过平均值 5 倍
- 批量异常检测
  - 交易重复检测
  - 成交量异常检测

涉及文件：

- `trade-strategy-ai/src/pipeline/validation.py`
- `trade-strategy-ai/src/pipeline/__init__.py`

## 当前设计决策

- 当前文章评论采用 JSON 快照方式存储在 `blog_articles.comments_payload`
- 当前不额外拆 `article_comments` 表，降低 Phase 1 的建模复杂度
- 当后续需要做评论级别线程分析、用户画像、评论关系图谱时，再拆出独立评论表
