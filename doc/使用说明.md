# 使用说明

## 本地安装（无 Docker）

前置：Python 3.11+。

推荐在 workspace 根目录创建统一虚拟环境，然后以 editable 方式安装：

```bash
cd ..
python -m venv .venv
source .venv/bin/activate

cd trade-strategy-ai
pip install -e ".[dev]"
```

## 淘股吧 Cookie 配置

第一版淘股吧抓取需要手工提供登录后的 Cookie。

### 获取方法

1. 在浏览器中登录淘股吧
2. 打开开发者工具
3. 进入 `Network`
4. 刷新一个登录后可访问的淘股吧页面
5. 选择任意请求，找到 `Request Headers`
6. 复制 `Cookie` 整段内容

### 推荐配置方式

推荐通过环境变量注入，再在配置文件中引用：

```bash
export TGB_COOKIE='你的完整 Cookie'
```

然后在 `trade-strategy-ai/config/app.yaml` 中配置：

```yaml
crawl:
  auth:
    tgb.cn:
      mode: cookie
      cookie: "${TGB_COOKIE}"

  # 抓取来源列表（与 auth 分离；支持同站点多作者）
  sources:
    - source: tgb
      site: tgb.cn
      trader_id: trader_a
      author_id: "10461311"
      author_name: "javxsp"
      list_url: "https://www.tgb.cn/user/blog/moreTopic?userID=10461311"
      enabled: true
    - source: tgb
      site: tgb.cn
      trader_id: trader_a
      author_id: "20000000"
      author_name: "another_user"
      list_url: "https://www.tgb.cn/user/blog/moreTopic?userID=20000000"
      enabled: true
```

也可以直接写入配置文件：

```yaml
crawl:
  auth:
    tgb.cn:
      mode: cookie
      cookie: "你的完整 Cookie"

  sources:
    - source: tgb
      site: tgb.cn
      trader_id: trader_a
      author_id: "10461311"
      author_name: "javxsp"
      list_url: "https://www.tgb.cn/user/blog/moreTopic?userID=10461311"
      enabled: true
```

不推荐直接把真实 Cookie 提交到仓库。

### 使用方式

抓取程序会将该 Cookie 作为 HTTP 请求头中的 `Cookie` 使用，用于访问登录后可见的列表页、文章页和评论页。

同一个站点下的多个作者可以共用同一份域名认证配置；未来如果接入其他站点，则在 `crawl.auth` 下增加新的域名项，并在 `crawl.sources` 中增加对应站点来源即可。

程序约束：

- 不在业务 JSONL 产物中保存 Cookie
- 日志中应避免输出完整 Cookie
- 当出现未登录、验证码、`403`、`429` 时应终止或退避

# 环境变量设置与一键运行说明

## 1. 设置淘股吧 Cookie 环境变量

强烈建议用环境变量注入 Cookie，避免明文写入配置文件：

```bash
export TGB_COOKIE='你的完整 Cookie'
```

可写入 ~/.zshrc、~/.bashrc 或直接在当前 shell 执行。

## 2. 生成配置文件（如首次使用）

```bash
cd trade-strategy-ai
python -m cli.main init-config
# 按需编辑 config/app.yaml，确保 crawl.auth.tgb.cn.cookie: "${TGB_COOKIE}"，并配置好抓取作者
```

## 3. 一键运行盘前/盘后流程

确保已激活 Python 虚拟环境（如 .venv）：

```bash
source ../.venv/bin/activate  # 路径按实际情况调整
```

运行盘前/盘后：

```bash
python -m cli.main run-pre-market --config config/app.yaml
python -m cli.main run-after-close --config config/app.yaml
```

## 3.1 数据库 / 迁移 / Pipeline / 抽取 / 聚类（推荐流程）

准备本地 PostgreSQL（推荐本机安装；Docker 仅作为可选方案）：

macOS（Homebrew）示例：

```bash
brew install postgresql@15
brew services start postgresql@15
```

创建数据库与用户（示例：trade/trade）：

```bash
psql postgres -c "CREATE ROLE trade WITH LOGIN PASSWORD 'trade';"
createdb -O trade trade_strategy_ai
```

准备环境变量（建议复制一份到 `.env` 并按需修改；不要提交密钥）：

```bash
cp .env.example .env
```

异步连接可用性验证：

```bash
python -m cli.main db-check --config config/app.yaml
```

执行 Alembic 迁移：

```bash
python -m cli.main db-migrate --config config/app.yaml
```

一键跑通（crawl → clean → validate → store）：

```bash
python -m cli.main pipeline-run --config config/app.yaml
```

LLM 抽取 v0（未配置 LLM 时会 fallback 并记录 raw_llm_output）：

```bash
python -m cli.main extract-articles --config config/app.yaml --limit 10
```

从真实抽取数据生成 StyleClusters：

```bash
python -m cli.main clusters-build --config config/app.yaml
```

端到端回归（crawl→store→extract→clusters→run-pre-market+HTML）：

```bash
python -m cli.main e2e-regression --config config/app.yaml
```

如需导出 HTML 日报/考核报告：

```bash
python -m cli.main run-pre-market --config config/app.yaml --export-html
python -m cli.main run-after-close --config config/app.yaml --export-html
```


## 4. 配置优先级说明

本项目支持多种配置方式，**优先级如下（高→低）**：

1. **命令行参数**（如部分命令的 `--database-url`，极少用）
2. **环境变量**（推荐，`.env` 文件或 shell 里 `export`，如 `DATABASE_URL`、`TGB_COOKIE` 等）
3. **配置文件 `config/app.yaml`**（如 `database.url`、`crawl.auth` 等）
4. **代码默认值**（兜底，不建议依赖）

**实际建议：**
- 数据库连接串（`DATABASE_URL`）：优先写在 `.env` 文件，或直接导出为环境变量。也可在 `config/app.yaml` 的 `database.url` 字段配置。
- Cookie、API Key 等敏感信息：优先用环境变量注入，YAML 里用 `${TGB_COOKIE}` 方式引用。
- 其他业务参数（如抓取作者、调度时间等）：只需在 `config/app.yaml` 里配置。

**举例：**

- `.env` 文件（推荐）：

  ```env
  DATABASE_URL=postgresql+asyncpg://trade:trade@localhost:5432/trade_strategy_ai
  TGB_COOKIE=你的cookie
  ```

- `config/app.yaml` 也可写：

  ```yaml
  database:
    url: "postgresql+asyncpg://trade:trade@localhost:5432/trade_strategy_ai"
  ```

如有多处配置，**优先级高的会覆盖低的**，无需重复设置。

## 5. 输出路径说明

所有产物（JSON/HTML/聚类/路由等）默认输出到

```
trade-strategy-ai/data/processed/phase0/
```
（可通过 config.yaml 的 storage.output_dir 配置）

## 5. 常见问题与注意事项

- Cookie 失效、403/429、验证码等会导致抓取中断，请及时更新 Cookie。
- 不要把真实 Cookie/API Key 提交到代码仓库。
- 环境变量优先级高于配置文件明文。
- 盘前/盘后命令可单独运行，也可写入定时任务。
