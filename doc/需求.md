# 📊 AI 交易策略反推与 Agent 系统项目文档（需求）

---

# 一、项目概述（Project Overview）

## 🎯 项目目标

构建一个完整的 AI 系统，实现：

> **从多个交易员的文章 + 交易记录 → 生成多个独立交易员 TraderAgent → 盘前给出交易标的建议 → 盘后复盘考核 → 复盘结论反馈到后续交易中，形成可解释的持续改进闭环**

---

## 🧠 核心能力

系统最终具备：

- 📚 持续获取并理解交易员文章（增量更新）
- 📈 分析交易员交易记录（真实操作）
- 🤖 为每个交易员生成独立 TraderAgent（交易习惯/风格隔离）
- 📊 盘前生成交易标的建议（买入价/目标价/止盈止损/仓位/失效条件）
- 🧾 Manager 汇总生成日报，并在盘后对建议进行考核与复盘
- 🔁 对收益不达标或亏损的交易员触发复盘总结并写回“记忆”，用于后续改进
- ⚙️（阶段性能力）策略反推/对齐/DSL/回测作为 TraderAgent“画像与规则”的上游输入

---

## 🚀 最终产出

- 一个 **多交易员 Agent 编排系统**（DataAgent + ManagerAgent + N 个 TraderAgent）
- 一套 **可扩展数据能力**（以 skill 方式扩展 DataAgent 能力）
- 一套 **盘前建议 + 盘后考核复盘** 的日常闭环（时间可配置，支持手动运行）
- 一个 **可解释、可回滚的持续改进机制**（记忆/复盘反馈驱动，不依赖 GPU 训练）
- （中后期）策略反推引擎、行为建模、对齐分析、DSL 执行与回测验证能力

# 二、项目需求（Requirements）

## 🆕 需求变动摘要（2026-04-03）

你会提供多个交易员的文章与交易记录：

1. 每个交易员最终变成一个独立 TraderAgent
2. 系统包含：
  - DataAgent：提供数据能力（接口可不断增加）
  - ManagerAgent：编排、汇总、考核、触发复盘任务，负责每日分析报告生成与HTML展示，支持后续Web/静态浏览扩展
3. 每天盘前（时间可配置，可手动）
  - TraderAgent 根据交易习惯向 DataAgent 请求数据
  - DataAgent 返回对应数据；若提供不了，向 ManagerAgent 提出待办任务（Task）
  - TraderAgent 输出结构化交易标的建议，并提交给 ManagerAgent
4. 每天收盘后（时间可配置，可手动）
  - ManagerAgent 基于最新股票数据对当天建议进行汇总与考核
  - 对收益不达标或亏损的 TraderAgent 触发复盘分析与总结，并反馈到后续交易
5. 文章会不定期更新：需要持续增量获取机制
6. 无 GPU：使用第三方大语言模型 API（OpenAI/Anthropic 等）

最终目标：实现一个不断学习、不断进步且可解释的交易系统。

---

## 🧩 可行性方案与运行形态（推荐）

在无 GPU 且需要持续运行的前提下，推荐采用“双模式同核”的落地方式：

1. 交互式模式（类 Claude Code）
  - 通过 CLI/脚本手动触发：抓文/盘前建议/盘后复盘
  - 优点：迭代快、便于调试提示词与数据能力、成本低
  - 适用：Phase 0-1 的闭环验证与快速试错

2. 独立程序长期运行模式
  - 作为独立服务运行：FastAPI（手动触发/查询） + scheduler/worker（按配置定时跑批） + DB（存储数据与报告）
  - 优点：稳定、可观测、可扩展到 N 个交易员
  - 适用：Phase 1-3 稳定化后长期运行

两种模式共享同一套核心应用层（Agent、契约、数据访问与评估逻辑），避免“交互式一套、服务一套”的维护分叉。

---

## 🔗 Agent 间调用与管理（最简一致协议）

- ManagerAgent 负责编排：决定盘前/盘后任务何时执行（来自配置），以及调用哪些 TraderAgent
- TraderAgent 只通过结构化契约与 DataAgent/ManagerAgent 通信：
  - 请求数据：DataRequest → DataResponse
  - 上报建议：TradeIdea
  - 接收复盘：EvaluationRequest → EvaluationResult
- DataAgent 统一对外提供数据接口；内部以 skills 扩展能力；能力缺失通过 `capability_missing` 回传并生成待办 Task

# 四、Agent 与 Skills 设计

## 🤖 0. Manager Agent（编排与考核，新增核心）
**职责**
- 盘前/盘后定时编排（时间来自配置，支持手动触发）
- 汇总 TraderAgent 的 TradeIdea 生成日报
- 收盘后基于最新数据对建议做考核评分，生成复盘报告
- 对收益不达标或亏损的 TraderAgent 触发复盘任务，并把结论写回其记忆
- 当 DataAgent 缺能力时，将 capability_missing 转化为可追踪 Task

**能力/接口（示例）**
- `run_pre_market()` / `run_after_close()`
- `collect_trade_ideas()` / `generate_daily_report()`
- `evaluate_trade_ideas()` / `trigger_evaluation_tasks()`

---

## 🤖 0.1 Trader Agent（每交易员独立，新增核心）
**职责**
- 根据自身交易习惯向 DataAgent 请求数据（DataRequest）
- 生成结构化 TradeIdea（标的、买入价/触发、目标价、止盈止损、仓位、失效条件等）
- 接收 Manager 的复盘任务，给出分析与可执行改进点，并写回记忆

---

## 🤖 1. Data Agent（数据采集）
**职责**
- 采集博客、交易记录、市场数据
- 统一对外暴露 DataRequest/DataResponse 接口
- 以“skills 注册表”方式持续扩展数据能力
- 无法提供数据时返回 capability_missing，并向 Manager 提出 Task

**Skills**
- `crawl_blog`
- `crawl_dynamic_page`
- `parse_html`
- `extract_trade_log`
- `fetch_market_data`
- `store_db`
- （可扩展）`fetch_index_data` / `fetch_sector_data` / `compute_indicators` / `fetch_fundamental_data` 等

---

## 🤖 2. Knowledge Agent（文章理解）
**职责**
- 将文章转换为策略结构

**Skills**
- `extract_concepts`
- `extract_rules`
- `extract_preconditions`
- `build_strategy_dsl`

---

## 🤖 3. Behavior Agent（行为分析）
**职责**
- 分析交易行为

**Skills**
- `label_trade_behavior`
- `feature_extraction`
- `pattern_mining`
- `clustering`

---

## 🤖 4. Alignment Agent（⭐核心）
**职责**
- 对齐“文章策略 vs 实际行为”

**Skills**
- `rule_match_score`
- `behavior_fit_score`
- `conflict_detection`
- `confidence_scoring`

---

## 🤖 5. Strategy Agent（决策）
**职责**
- 生成买卖信号

**Skills**
- `compute_features`
- `evaluate_rules`
- `combine_scores`
- `generate_signal`

---

## 🤖 6. Risk Agent（风控）
**职责**
- 控制风险

**Skills**
- `position_sizing`
- `stop_loss`
- `drawdown_control`

---

## 🤖 7. Backtest Agent（回测）
**职责**
- 验证策略

**Skills**
- `run_backtest`
- `evaluate_metrics`
- `parameter_search`

---
